🦖 Dino WCAG Oefening – ReadMe
Introductie

Welkom bij de Top 5 Dino’s website oefening!
In deze opdracht ga je een bestaande webpagina analyseren, klonen naar je eigen project en verbeteren zodat deze WCAG AA-proof wordt.

De site is opzettelijk niet volledig toegankelijk. Het doel is dat jij leert herkennen waar problemen zitten en hoe je deze oplost.

Doelen van de oefening
Identificeer WCAG-problemen in de header, main content en footer.
Pas semantiek, kleuren, contrast en keyboard interacties aan.
Maak de pagina responsive voor verschillende schermgroottes.
Oefen met alt-teksten, focus states, en semantische HTML.
Begrijp hoe kleine CSS- of HTML-fouten grote gevolgen kunnen hebben voor toegankelijkheid.

Opdracht
Kloon de huidige site naar je lokale project.
Analyseer alle WCAG-problemen met een checklist of browser plugin (bijv. Lighthouse, axe, Contrast Checker).
Los problemen op:
Verbeter semantische HTML (<header>, <article>, <section>, <footer>).
Voeg alt-teksten toe bij alle afbeeldingen.
Verhoog contrast tot AA-standaard.
Zorg dat buttons en klikbare kaarten keyboard toegankelijk zijn.
Verbeter focus-states en hover-states.
Maak de pagina responsive voor tablet en mobiel.
Test je verbeteringen met een screenreader en op meerdere apparaten.
Tips
Gebruik kleurcontrast checkers: dit helpt subtiele fouten op te sporen.
Denk aan cognitieve toegankelijkheid: korte zinnen, duidelijke structuur.
Focus op alles wat visueel werkt en alles wat met toetsenbord/screenreader werkt.
Kleine CSS hacks kunnen grote gevolgen hebben voor toegankelijkheid → let op “tricky” kleuren en hover-effects.
Extra uitdaging
Voeg aria-labels of aria-describedby toe waar nodig.
Zorg dat de dino-kaarten klikbaar en toegankelijk zijn via keyboard, niet alleen muis.
Experimenteer met alt-tekst die niet alleen zegt “dino afbeelding”, maar ook beschrijft hoe de dino eruitziet.
